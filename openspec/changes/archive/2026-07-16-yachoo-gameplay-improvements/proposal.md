## Why

현재 디스코드 야추 게임 서비스는 진행 중인 대전이 있음에도 중복 대전이 무한 생성되어 게임 진행 상태가 꼬이는 문제와 모바일/저해상도 화면에서 주사위 홀드 버튼의 텍스트 너비 변화로 UI 레이아웃이 깨지는 문제를 겪고 있습니다. 또한, 기권 확인 단계 부재로 오클릭이 빈번하게 일어나며, Yacht(야추) 족보 완성 시의 알림이 밋밋하거나 이미 야추를 포기한 유저가 뒤늦게 야추를 완성했을 때의 재미 요소(놀림)가 부족합니다. 본 변경안을 통해 게임의 구조적 결함을 보완하고 한국 인터넷 커뮤니티 톤앤매너에 맞게 게임 내 상호작용의 몰입감과 재미를 강화하고자 합니다.

## What Changes

- **중복 대전 제한**: 동일 플레이어 조합에 대해 진행 중인(InProgress) 게임은 하나만 활성화되도록 제한하고, 중복 신청 시 기존 게임의 Discord 메시지로 바로갈 수 있는 포워딩 링크를 에페메랄(비공개) 메시지로 전송합니다.
- **D1 데이터베이스 최적화**: 완료된 게임(Finished)은 `active_games` 테이블에서 즉시 삭제(DELETE)하도록 라이프사이클을 최적화하고, 기존 Finished 데이터들을 일괄 정리하는 마이그레이션을 진행합니다. 하위 호환성을 위해 InProgress 필터 조건은 유지합니다.
- **점수판 레이아웃 개선**: 점수판 모노스페이스 뷰에서 서브토탈과 보너스 영역 위아래에 단일 수평 바(`==========================`) 형태의 구분선을 삽입합니다.
- **기권 프로세스 안전장치**: 기권 버튼 라벨 텍스트를 제거하고 깃발 이모지(`🏳️`)만 표기해 버튼 너비를 축소하며, 클릭 시 "정말 기권하시겠습니까?"를 묻는 에페메랄 확인용 버튼 메시지를 노출합니다.
- **Yacht 관련 어그로 및 놀림 시스템**: Yacht(50점) 기록 시 남초 커뮤니티(디시, 펨코 등) 톤의 어그로성 축하 메시지를 채널에 박제하며, 이미 Yacht를 0점 처리했거나 소진한 상태에서 뒤늦게 야추 눈금을 완성한 경우 조롱 메시지를 박제합니다. 그 외 연속 0점, 최종 꼴등에 대해서도 동일한 톤의 놀림 메시지 풀을 추가합니다.
- **주사위 버튼 너비 고정**: 주사위 홀드 여부에 따라 버튼 크기가 변경되어 레이아웃이 출렁거리지 않도록, 일반 상태 `[1]`과 Held 상태 `🔒`의 물리적 폰트 너비가 대등하도록 라벨 구조를 고정합니다.

### Modified Capabilities
- `game-orchestrator`: 대전 매칭 시 중복 게임 검증 및 기존 대전 메시지 포워딩 링크 생성을 처리하도록 조율 기능 변경.
- `d1-database-schema`: active_games 테이블의 게임 완료 시 삭제 처리와 기존 데이터 정리를 위한 마이그레이션 적용.
- `yacht-score-rules`: Yacht 달성 성공 및 실패(헛박제) 시의 채널 알림 조건 및 추가 놀림 시나리오(낮은 점수, 연속 0점, 최종 꼴찌) 트리거 적용.

## Impact

- **UI / Serialization**: `src/presentation/discord/adapter/serializer.ts` (점수판 렌더링, 버튼 라벨, 이모지, 기권 버튼 축소, 메시지 템플릿 처리)
- **Engine / Actions**: `src/index.ts` (인터랙션 핸들러 - 중복 게임 세션 존재 검사, 메시지 포워딩 구성, 에페메랄 기권 확인 분기, 야추/놀림 채널 알림 발송 로직)
- **Persistence**: `src/persistence/d1/repository.ts` 및 `src/persistence/repository.ts` (GameRepository에서 완료 시 DELETE 메소드 추가, InProgress 대전 여부 체크 추가)
- **Database Schema**: `migrations/` 내 신규 SQL 파일 추가 (기존 Finished 데이터 삭제 및 active_games 쿼리 호환성)
