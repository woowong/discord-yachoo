## Context

야추 다이스의 웹 대시보드와 데이터 수집 로직에 버그가 존재하여 수정이 필요하며, 사용자가 매치 모드에서 ELO 변화량을 확인할 수 있도록 그래프를 도입하고자 합니다.
기존 데이터베이스에는 플레이어별 현재 ELO 등급만 저장되어 있어 과거 시점의 ELO 변동 히스토리를 파악할 수 없었습니다. 따라서 이를 경기(Match) 이력과 함께 영속화하고 프론트엔드단에서 차트로 표현하기 위한 설계가 필요합니다.

## Goals / Non-Goals

**Goals:**
- 경기 복기 모달 내 닉네임 클릭 연동 오류 해결 (프로필 탭 전환 및 로딩).
- `guildId`가 없을 때의 조회 조건을 완화하여 전적/평균 점수 전체 노출 보장.
- 복기 모달 테이블에 누적 점수 추적, 플레이어 간 시각적 배경/아이콘 분리, 점수별 하이라이트 추가.
- `matches` 테이블 스키마에 경기 결과 시점 ELO 상태를 기록하는 마이그레이션 생성.
- 웹 프로필 대시보드에 Chart.js 기반의 ELO 변동 반응형 라인 차트 구현.

**Non-Goals:**
- 디스코드 봇 인터페이스 내 ELO 그래프 이미지 생성 및 렌더링.
- 기존 완료된 모든 경기에 대한 과거 ELO 가상 시뮬레이션 복원 (기존 매치는 NULL로 처리).
- 핵심 야추 다이스 엔진 도메인 비즈니스 로직 수정.

## Decisions

### 1) ELO 기록 방식 (Database Schema)
- **결정**: `matches` 테이블에 `player1_elo_after` 및 `player2_elo_after` 컬럼을 정수형(INTEGER)으로 추가합니다.
- **근거**: 매치 자체가 고유 ID와 함께 시간 순으로 적재되는 로그 형식이므로, 별도의 변동 히스토리 테이블을 설계하는 것에 비해 스키마 구조가 단순해지고 쿼리가 최적화됩니다.
- **대안**: `elo_history` 독립 테이블 신설. 정규화 측면에서 우수하나, 매치 데이터 조회 시 조인 연산 비용이 추가되고 관리가 복잡해집니다.

### 2) ELO 선계산 및 워크플로우 순서 변경
- **결정**: `GameWorkflowService.ts`에서 경기 종료 처리 시 `saveMatch` 이전에 ELO 변경 점수를 선계산하고, 이 값을 `MatchRecord` 객체에 바인딩하여 데이터베이스에 한 번에 저장합니다.
- **근거**: 기존에는 `saveMatch`를 먼저 호출하고 봇 메시지를 가공하는 과정에서 ELO를 조회/계산하여 `updateElo`를 별도로 수행했습니다. 선계산 프로세스로 일원화하면 ELO 상태값의 일관성이 향상되고, 불필요한 데이터베이스 조회 연산이 감축됩니다.

### 3) 길드 식별값 미제공 시의 DB 조회 조건
- **결정**: 리포지토리의 `getRecentMatches` 및 `getPlayerAverageScore` 쿼리에서 `guildId` 매개변수가 `null`인 경우, `guild_id IS NULL` 조건부 필터를 완전히 생략하고 쿼리합니다.
- **근거**: 싱글 모드는 길드가 없을 수 있으나 매치 모드는 길드 내에서만 수행됩니다. `guildId`가 전달되지 않는 대시보드 메인 진입 상태에서는 플레이어의 모든 활동 이력과 전체 평균 점수를 종합하여 렌더링하는 것이 요구사항에 부합합니다.

### 4) 차트 라이브러리 선정
- **결정**: `Chart.js` 라이브러리를 CDN(`cdn.jsdelivr.net`)을 통해 단일 HTML 템플릿의 `<head>` 영역에 로드합니다.
- **근거**: 별도의 빌드 도구나 웹팩 설정 없이 경량 서버리스 워커 환경에서 동작하는 단일 파일 대시보드 구조에 적합하며, 풍부한 Line chart 커스터마이징 옵션을 제공합니다.

## Risks / Trade-offs

- **[Risk] ELO 누락**: 기존 매치 레코드들은 신규 ELO 컬럼 값이 `NULL`로 들어가므로 그래프 렌더링에 예외가 생길 수 있습니다.
  - **Mitigation**: 프론트엔드 가공 로직에서 ELO 값이 존재(`IS NOT NULL`)하는 멀티플레이 매치만 필터링하여 그래프 포인트로 매핑합니다. 데이터가 부족하거나 없는 경우 차트 영역을 숨깁니다.
